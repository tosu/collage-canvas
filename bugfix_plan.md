# Bugfix Plan: 背景色对比不足导致视觉“抖动/舞动”

## 现象
- 左侧（sidebar 背景 #1a1a1a）与右侧主区域背景 (#121214) 对比极弱，整体像一块色块“抖动”且层次不清，分区不明确。

## 目标
- 提升左右分区的层次感和稳定感：清晰的明暗/色相差异，但不过度跳脱；保持现有玻璃化/暗色风格。
- 保持可读性与对比度：文本、控件依然清晰，无眩光。

## 方案原则
- 适度提高明暗差：使 sidebar 背景比主区域略亮/冷，主区域更深或带轻微纹理。
- 加分隔线/阴影：强化分区，不依赖大面积色差。
- 统一色系：仍在蓝紫/深灰体系内，避免杂色。

## 配色建议
- Sidebar 背景：`#181a20`（比主区域亮、偏冷蓝），或 `#161822`。
- 主区域背景：`#0e0f14` 或 `#0c0d12`；可叠加微弱渐变/噪声纹理。
- 分隔线：`rgba(255,255,255,0.07)`；玻璃边框同调。
- Hover/强调色不变（主色 #6366f1），确保对比够高。

## 实施步骤
1) 变量调整
   - 更新 `--sidebar-bg`、`--bg-color`（或新增 `--main-bg`）以体现层次差异。
   - 若需，添加轻微背景渐变/纹理到主区域（CSS background-image）。
2) 分隔强化
   - Sidebar 右侧边框/阴影加强：`border-right`/`box-shadow` 适度提升。
3) 验收
   - 视觉上左右分区清晰稳定，无“舞动”感；文字与控件对比良好。
